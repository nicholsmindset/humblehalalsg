## 4. Vendor badge / cross-link program

Uses the existing badge generator at `/for-business/badge` (free "Verified Halal / Listed on Humble Halal" embed that links back to the vendor's listing).

**Why it works:** wedding vendors (caterers, pelamin, photographers, bridal) mostly run their own WordPress/Wix sites and Instagram — real, editorially-controlled domains. 10–15 badge placements from genuinely listed vendors is clean, relevant link equity.

**Sequence for the 20 seeded wedding vendors:**

| Step | When | Action |
|---|---|---|
| 1. Warm-up | Week 1 | Verify each vendor's listing is accurate first (photos, packages, contact). A wrong listing kills the ask. |
| 2. Email 1 | Week 1–2 | Template E. Personalise line 1 with their actual category page. 5 vendors/week so replies stay manageable. |
| 3. Listing-update hook | On reply | Do the update within 24h, then: "Done — and the badge is here if you want it: /for-business/badge." Reciprocity does the work. |
| 4. Follow-up | +7 days, no reply | One short bump: "Any updates you'd like on your listing?" Never re-ask for the badge in the bump. |
| 5. WhatsApp/IG fallback | +14 days | Many vendors live on WhatsApp/Instagram, not email. One polite DM max. |
| 6. Install help | On interest | Offer to send the exact embed snippet or add it for them (Wix/WordPress instructions). Most "yes but never installed" cases are technical friction. |
| 7. Log + showcase | Ongoing | Track installs (Section 5). Once 5+ live, add a "vendors displaying our badge" row to the badge page for social proof. |

**Rules:** badge is free and stays free, no follow/nofollow demands, no anchor-text requests — the badge's default link is enough. If a vendor asks to be paid for placement, decline (that's a bought link).

**Expected yield:** 20 contacted → ~10 replies → 5–8 badge installs in the first cycle. Repeat with each new vendor batch onboarded.

### What NOT to do (non-negotiable)

- **Never buy links.** No paid guest posts, no "sponsored without disclosure", no link sellers, no Fiverr/PBN packages. The domain is recovering from a disavowed toxic PBN profile — one paid-link pattern re-poisons the profile and wastes the disavow.
- **No link exchanges at scale.** The mall swap (template D) is a genuine editorial partnership; a page of reciprocal "partners" links is not. Cap reciprocal arrangements and keep each one defensible.
- **No mass email blasts.** BCC'ing 25 outlets gets the domain flagged and burns every relationship at once.
- **No fake data.** Every number in a pitch must exist on the cited page. If the wedding ranges change on `/malay-wedding-singapore`, update this kit's template B first.
- **Never invent or scrape journalist emails.** Outlet-level channels only until someone replies personally.
- **Don't pitch MUIS** and never imply MUIS endorsement. Certification claims stay exactly as the site states them (deep-link, own assertion).
- **No widget-bait beyond the vendor badge.** The badge goes only to businesses actually listed; never distribute it as a generic "add this to your site" embed.
- **Don't chase irrelevant DR.** A DR 20 SG wedding blog beats a DR 60 offshore "guest post" site every time. Relevance first.
